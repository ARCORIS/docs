## Purpose

This page is the main entry point for ARCORIS documentation.

Use it to choose the right documentation layer, understand what each top-level section is responsible for, and move to the section that matches your goal.

## Start here

If you are new to ARCORIS, start with the [Start Here](start-here/index.md) section.

That section provides audience-based entry paths for:

- evaluators;
- integrators;
- operators;
- contributors.

If you already know the kind of material you need, use the section map below and move directly to the appropriate section.

## Reading order

A simple reading order for the documentation set is:

1. Start with [Start Here](start-here/index.md) if you need an audience-based entry path.
2. Continue to [Overview](overview/index.md) for system identity, scope, and problem framing.
3. Continue to [Concepts](concepts/index.md) for shared terminology and conceptual models.
4. Continue to [Architecture](architecture/index.md) for system structure, boundaries, and major flows.
5. Use [Tutorials](tutorials/index.md) and [Guides](guides/index.md) for practical learning and task-oriented work.
6. Use [Operations](operations/index.md), [Security](security/index.md), [Releases](releases/index.md), and [Reference](reference/index.md) for long-term operational and technical material.

## Section map

The documentation is organized into a small set of top-level sections.

| Section | Responsibility |
| --- | --- |
| [Start Here](start-here/index.md) | Entry paths for different reader types and first-step orientation |
| [Overview](overview/index.md) | System identity, scope, problem space, goals, assumptions, and high-level operating understanding |
| [Concepts](concepts/index.md) | Shared conceptual language needed to read the rest of the documentation correctly |
| [Architecture](architecture/index.md) | Canonical system structure, boundaries, major flows, and architectural explanation |
| [Tutorials](tutorials/index.md) | Step-by-step learning material for first runs and first practical exploration |
| [Guides](guides/index.md) | Task-oriented documentation for doing specific kinds of work |
| [Operations](operations/index.md) | Runtime operation, readiness, observability, sizing, incident handling, and support-facing material |
| [Security](security/index.md) | Public security documentation for deployment-facing and integration-facing security concerns |
| [Releases](releases/index.md) | Versioning, upgrades, migration, support windows, and release-facing documentation |
| [Reference](reference/index.md) | Stable technical reference material, terminology, compatibility, and generated reference entry points |

## Documentation model

Different sections serve different purposes.

- [Overview](overview/index.md) explains what the system is.
- [Concepts](concepts/index.md) explains the language needed to reason about it.
- [Architecture](architecture/index.md) explains how it is structured.
- [Tutorials](tutorials/index.md) and [Guides](guides/index.md) help readers learn and perform concrete tasks.
- [Operations](operations/index.md), [Security](security/index.md), [Releases](releases/index.md), and [Reference](reference/index.md) provide the durable material needed after the system baseline is understood.
